# Docker ネットワーク

- `docker network create [ブリッジ名]`
  - ユーザ定義のブリッジを作成
- `docker network connect [ブリッジ名] [コンテナ名]`
  - コンテナをブリッジに接続する
- 上記内容を行うことでコンテナ名で名前解決することができる。
  - ブリッジ内で独自のDNSによって名前解決される
- `docker network disconnect [ブリッジ名] [コンテナ名]`
  - ブリッジからコンテナを切り離す

## ノンネットワーク

- 他のネットワークのインタフェースを持たない

## ホストネットワーク

- Dockerホストと同じネットワークになる
  - IPアドレスがホストと同じになる
